# Docker-sandboxed Codex execution with mise-driven generic image

Codex phases run inside an ephemeral Docker container per Codex execution — **Planning Phase**, **Assignment Attempt** implementation, **Review Phase**, and **Merge Phase** — rather than directly on the host. The **Planning Phase** sandbox bind-mounts the **Project** path read-only since the planner does not edit project files; the other phases bind-mount the **Assignment Worktree** read-write. The **Codex Sandbox** isolates each agent execution's network so parallel **Issue Assignments** can bind the same port (dev servers, playwright fixtures, test databases) without colliding, while preserving outbound access for the Codex API call and `git push`. A single generic runtime image carries `codex`, `git`, `mise`, and a minimal base; container entrypoint runs `mise install` against the **Assignment Worktree**'s `mise.toml` so Project toolchain versions match the host's source of truth without per-Project Dockerfiles.

The container is named `agentic-afk-assignment-<assignment-uuid>` and labelled with the **Plan Run**, **Issue Assignment**, **Assignment Attempt**, **Project**, and phase identifiers. Labels exist so orphan sweep, dashboard surfacing, and `docker ps` filtering all work without parsing names. Containers run with `--rm` so happy-path exit removes them automatically. Stdout is captured live by the orchestrator and parsed for structured **Phase Output** exactly as ADR-0030 specifies; `docker logs` is not on the critical path.

The container runs with `--user $(id -u):$(id -g)` so files written into the bind-mounted **Assignment Worktree** stay owned by the host user, and OAuth refresh-token writes to `~/.codex/auth.json` land on the host with the correct owner. Resource caps `--memory=8g --cpus=2.0 --pids-limit=512 --restart=no` bound runaway-Assignment damage on a single-developer host. Default bridge networking provides outbound NAT and Docker's embedded DNS; no `--network host`, no `--publish` — the host never sees agent-spawned ports, matching the constraint that the developer does not need to watch in-flight dev servers from the host.

Auth uses the OAuth path for the first slice. `~/.codex/auth.json` is bind-mounted read-write into the container at `$HOME/.codex/auth.json` (where `$HOME` is set to a per-container temp dir to avoid colliding with image conventions). All concurrent **Codex Sandboxes** mount the same `auth.json`. Refresh-token rotation under parallel refresh is a known race; first-slice mitigation is treating the rare auth failure as transient and retrying the **Assignment Attempt** once before blocking. `OPENAI_API_KEY` env passthrough is deferred until a user asks for it. The Control Plane writes a per-Assignment Codex `config.toml` to a temp dir and bind-mounts it read-only at `$HOME/.codex/config.toml`, preserving ADR-0030's "Control Plane owns the profile/config overlay" rule.

Project toolchain comes from `mise.toml` in the **Assignment Worktree**. The image carries `mise` only; the entrypoint runs `mise install` before invoking `codex exec`. A named Docker volume `agentic-afk-mise-cache` mounted at mise's data dir persists tool installs across containers so the first Plan Run for a Project is the only slow one. The volume is shared across all **Projects** and **Codex Sandboxes** because mise content-addresses by version; concurrent `mise install` invocations rely on mise's own locking.

**Plan Run** preflight extends ADR-0028 with four sandbox checks at trigger time, before any claim writes: Docker daemon reachable; runtime image present or local build succeeds; **Project** worktree contains a `mise.toml`; `~/.codex/auth.json` exists and is readable. Any failure surfaces from the trigger without writing a claimed or blocked **Source Issue** state, matching the existing preflight contract.

Cleanup runs on three boundaries. Happy-path exit relies on `--rm`. Orchestrator-initiated stop (cancel, capacity eviction, **Issue Assignment** blocked mid-flight) issues `docker stop --time 10` followed by escalation to `docker kill -s 9` + `docker rm -f` after a 20-second hard cap. On orchestrator boot, a synchronous sweep lists containers carrying the `agentic-afk.plan-run-id` label: containers belonging to **Issue Assignments** still in a running state are killed and the **Issue Assignment** transitions to `Blocked` with new **Block Reason** `OrchestratorRestart`; containers with no live owner are removed. Adoption (re-attaching to in-flight stdout) is rejected for the first slice because durable **Phase Output** only exists at phase completion, so a mid-flight container has no resumable state.

Image lifecycle stays manual for the first slice. The runtime image is built locally from a `Dockerfile` checked into this repo; the orchestrator builds it on first Plan Run if absent. Tagging follows the source-tree commit that produced the Dockerfile so multiple SHAs may coexist. Neither images nor the `agentic-afk-mise-cache` volume are auto-pruned; the operator runs `docker image rm` or `docker volume rm agentic-afk-mise-cache` if disk pressure appears. A published-image distribution path (GHCR pull) is deferred until the tool ships beyond the original developer.

Project trust under ADR-0031 stays semantically unchanged: the developer authorises agent execution against this **Project**, and that authorisation now applies inside the **Codex Sandbox**. The sandbox bounds the agent's reach to the bind-mounted worktree, the mounted auth file, and outbound network; it does not weaken or strengthen the trust act. The container is full-autonomy within those bounds, matching ADR-0030's non-interactive contract.

## Considered Options

- **No isolation, accept port collisions.** Rejected because every **Issue Assignment** in this repo can plausibly start the dashboard dev server or the playwright fixture on hardcoded ports (`playwright.config.ts` binds `127.0.0.1:3637`). Second concurrent **Assignment** would fail to bind. With `Max Parallel Tasks` targeted at 2–3 and most work network-touching, collision is near-certain rather than rare.

- **Per-Project `exclusive_resources` lock that serialises network-using Assignments.** Rejected because in this repo virtually every **Issue Assignment** would claim the lock, degenerating to fully serial execution and defeating the **Plan Run**'s parallel-issue-work invariant. Useful only when network-heavy work is a minority of Assignments.

- **Per-command `unshare -Urn` wrapper for test/build commands the agent invokes.** Rejected because the agent picks tools ad-hoc — `dx serve`, custom test runners, testcontainer-style fixtures — and there is no single chokepoint to wrap. Requires a per-Project shim and breaks when the agent introduces a new tool.

- **Hand-rolled `unshare -Urn` + `slirp4netns` around the whole Assignment.** Considered as a podman/Docker-free alternative. Solves port collision with the same primitives Docker uses internally (~50 LoC orchestrator plus a `slirp4netns` binary dependency, no daemon, no group membership). Rejected because we already accept Docker as an acceptable dependency for this single-developer setup, and Docker buys tooling (`docker logs`, `docker inspect`, `docker stop`), a clean cleanup contract via labels, and a future path to per-Project images if reproducible-env ever becomes a product feature. The marginal complexity over slirp4netns is paid for by removing custom Linux plumbing from the orchestrator.

- **Podman rootless instead of Docker.** Rejected at operator request — the developer does not want podman installed on the host. Docker daemon and `docker` group membership were acceptable in exchange.

- **Per-Project Dockerfile.** Rejected for the first slice because the only **Project** today is this repo and its toolchain is fully described in `mise.toml`. A single generic image plus runtime `mise install` matches Project tooling without forcing every **Project** to maintain its own Dockerfile. The hybrid path (generic base + Project overlay) stays available as a future refinement when a second **Project** demonstrates a real need.

- **Multi-backend support (Opencode + Codex) in the first slice.** Out of scope; ADR-0030 already deferred multiple backends, and that decision stands. The **Codex Sandbox** is shaped around the Codex contract but is not specific to it; if a second backend is introduced later, the sandbox primitives (label set, lifecycle, auth-mount convention) apply to it unchanged.

- **Adopt in-flight containers on orchestrator restart.** Rejected because **Phase Output** is only durable at phase completion (ADR-0030), so a mid-flight container has no state worth resuming. The simpler kill-and-block path is honest about that and produces a visible `Block Reason::OrchestratorRestart` rather than silent loss.

- **Auto-prune of images and the mise cache volume.** Rejected for the first slice. Image rebuilds are rare (the runtime image is tiny and stable), and the mise cache is content-addressed and shared across Projects, so disk pressure should be slow. Manual `docker volume rm` is a sufficient reset escape hatch. Auto-prune can be revisited if real operators report bloat.

- **`OPENAI_API_KEY` env passthrough alongside OAuth mount.** Deferred. The operator uses OAuth; supporting both adds a config-shape branch in preflight and the trust UI for no immediate user. Trivial to add later behind a single auth-method enum.
